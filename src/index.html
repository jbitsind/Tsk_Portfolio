Taxi Fare Prediction (GCP + XGBoost)

Cloud Computing capstone (CMU-Africa) — Oct 2024 to Dec 2024
Delivered an end-to-end, reproducible pipeline for taxi-fare estimation using NYC taxi records and GCP-hosted services. The project focused on robust preprocessing, strong feature engineering, model tuning, and low-latency deployment for online predictions.

  • Data & Preprocessing: Ingested and cleaned large-scale NYC taxi datasets (pickup/dropoff timestamps, coordinates, passenger counts, fares). Implemented validation checks, outlier filtering, and scalable feature pipelines to ensure reproducible training and evaluation.
  • Feature Engineering: Built time-based (hour, day-of-week), spatial (H3/geohash bins, distance, bearing), and contextual features (passenger count, estimated traffic proxies). Engineered features to reduce bias and improve generalization across boroughs.
  • Modeling: Trained and tuned an XGBoost model with cross-validation and hyperparameter search. Used careful validation splits to avoid temporal leakage and to measure realistic generalization performance.
  • Deployment: Packaged the model and serving logic in a Flask microservice and deployed to Google App Engine for horizontally scalable, low-latency inference (production latency target < 200 ms).
  • GCP Tools: Used AI Platform for managed training experiments, Cloud Storage for dataset and model artifacts, and App Engine for serving. Integrated logging and basic monitoring to track request rates and latencies.
  • Extras: Added landmark recognition using Cloud Vision API to enhance contextual routing and optional AutoML prototypes for rapid experimentation and model comparisons.
  • Reproducibility & Pipeline Integration: Automated preprocessing and model training steps so experiments can be re-run end-to-end; packaged artifacts with versioning to support retraining and audits.

Skills & tools: GCP (AI Platform, App Engine, Cloud Storage), XGBoost, Flask, Cloud Vision API, AutoML, Python, feature engineering, and pipeline integration.

Developing a Scalable Machine Learning-Based Potential Energy Surface (PES) Fit for Molecules: Sum-of-Product Approach

ICTP-EAIFR, Kigali, Rwanda

Developed Python software to construct machine-learned Potential Energy Surfaces (PES) in a sum-of-product representation for grid-based quantum-dynamical simulations, targeting spectroscopic-quality fits (RMSE ≈ 1 cm⁻¹) while maintaining scalability and symmetry awareness.

Approach

Investigated Atomic Cluster Expansion (ACE), Message-Passing Neural Networks, Kernel Ridge Regression, and Gaussian Process Regression; explored force-field-like functional forms and evaluated fits via quantum dynamics tests on Van der Waals, large-amplitude, and semi-rigid molecules.

Skills & Tools

Machine Learning, Python, TensorFlow, Model Optimization, Cluster Computing, Computational Chemistry, Neural Networks

Highlights

  • Constructed ML-based SOP PES representations and compared NN/KRR/GPR approaches.
  • Explored HDMR and symmetry-aware techniques to improve scalability.
  • Validated fitted PES within quantum-dynamical simulations to assess spectroscopic accuracy.
